# Техническое задание

`inventory_logs.txt` — лог предметов  
`money_logs.txt` — лог денег  

### Типы действий (action_type):
- `ITEM_ADD` — предмет получен  
- `ITEM_REMOVE` — предмет потерян  
- `MONEY_ADD` — деньги получены  
- `MONEY_REMOVE` — деньги потеряны  

### Формат логов предметов:
`[timestamp] action_type | player_id, (item_type_id, amount, item_type_id, amount, ...)`

### Формат логов денег:
`timestamp|player_id|action_type,amount,reason`

### Задача
Написать программу на **Python 2.7.18**, используя **только стандартные библиотеки**.

### Программа должна:

1. **Сформировать объединённый лог** `combined_log.txt`  
   Записи из обоих файлов объединяются и сортируются по времени.  
   При одинаковом времени — сначала записи из `inventory_logs.txt`.

   **Формат записей в combined_log.txt:**
    
    `[YY-MM-DD HH:MM:SS] player_id | ITEM_ADD (item_type_id, amount) (item_type_id, amount) ...`  
    `[YY-MM-DD HH:MM:SS] player_id | ITEM_REMOVE (item_type_id, amount) (item_type_id, amount) ...`  
    `[YY-MM-DD HH:MM:SS] player_id | MONEY_ADD | amount | reason`  
    `[YY-MM-DD HH:MM:SS] player_id | MONEY_REMOVE | amount | reason`  


2. **Вывести в** `output.txt` **статистику:**

- **Топ 10 предметов** по количеству упоминаний в логах (ADD + REMOVE).  
`Название предмета, количество`
- **Топ 10 игроков** по итоговому количеству денег после всех операций.  
`Имя игрока, кол-во денег, дата первого упоминания, дата последнего упоминания`
- **Первые 10 предметов**, которые появились в логах (в порядке появления).  
`Название, дата`
- **Последние 10 предметов**, которые появились в логах (в порядке появления).

3. **Интерактивный режим**

    После вывода статистики программа ждёт ввод `item_type_id` и для каждого запроса выводит:
   - Название предмета  
   - Общее количество таких предметов в наличии в игре
   - Общее количество игроков, у которых есть хотя бы один такой предмет
   - Топ 10 игроков по количеству данного предмета:
   - Имя игрока, кол-во предметов такого типа

### Примечания и ограничения
- 1 ≤ `item_type_id` ≤ 1000  
- 1 ≤ `player_id` ≤ 100000  
- Даты выводить строго в формате `[YY-MM-DD HH:MM:SS]`  
- Обязательно реализовать **класс игрока**  
- Деньги могут быть отрицательными  
- До **10 000 000** строк в логах суммарно  
- Решение должно быть **оптимальным** по времени и памяти  
- Сдавать в виде **.zip-архива**: исходные логи + код + combined_log.txt + output.txt + (по желанию) README
